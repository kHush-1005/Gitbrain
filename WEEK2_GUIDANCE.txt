WHAT CHANGED FROM WEEK 1 TO WEEK 2
────────────────────────────────────────────────────────────────────────────────

Week 1 built a working pipeline that:
  → Fetched files from GitHub
  → Chunked Python files using AST

  LAYER 1: GitHub Ingestion (github_ingester.py)

    + NEW: char_count and token_estimate fields on every chunk
    + NEW: repo_name field embedded in chunks when provided

    + CRLF line ending normalization

  LAYER 3: Data Persistence (NEW)
    + utils/file_utils.py — save and load JSONL and JSON files
    + scripts/export_chunks.py — full pipeline script that saves to disk
    + data/chunks/ directory — where JSONL files are stored

  LAYER 4: Quality Assurance (NEW)
    + scripts/validate_chunks.py — 8 automated checks on the JSONL file
    + tests/test_chunker.py — 25 pytest tests for the chunker
    + tests/test_repo_parser.py — 15 pytest tests for URL parsing

HOW TO RUN WEEK 2 — STEP BY STEP
────────────────────────────────────────────────────────────────────────────────

  ┌──────────────────────────────────────────────────────────────────────────┐
  │  STEP 1: Run the quick test (same command as Week 1, works as-is)        │
  └──────────────────────────────────────────────────────────────────────────┘

  python test_ingester.py https://github.com/psf/requests

  What it does:
    → Parses URL
    → Fetches files from GitHub API
    → Chunks all files with updated chunker (now includes JS heuristic)
    → Prints 20 sample chunks with char_count and token_estimate
    → Shows Week 2 validation summary

  Optional: save chunks from test_ingester.py directly:
    python test_ingester.py https://github.com/psf/requests --save


  ┌──────────────────────────────────────────────────────────────────────────┐
  │  STEP 2: Export chunks to disk (the main Week 2 script)                  │
  └──────────────────────────────────────────────────────────────────────────┘

  python scripts/export_chunks.py https://github.com/psf/requests

  What it does:
    → Fetches all files from GitHub (up to 500 by default)
    → Chunks every file
    → Saves result to: data/chunks/psf__requests_chunks.jsonl
    → Saves summary to: data/chunks/psf__requests_summary.json
    → Prints chunk type breakdown, language breakdown, size statistics

  For a larger repo with more files:
    python scripts/export_chunks.py https://github.com/tiangolo/fastapi --max-files 300

  After this step, open data/chunks/ and you should see the JSONL file.
  JSONL = JSON Lines = one JSON object per line. Open it in any text editor.


  ┌──────────────────────────────────────────────────────────────────────────┐
  │  STEP 3: Validate the JSONL file                                         │
  └──────────────────────────────────────────────────────────────────────────┘

  python scripts/validate_chunks.py data/chunks/psf__requests_chunks.jsonl

  What it checks:
    1. Required fields present on all chunks
    2. No chunks with empty text
    3. Valid line numbers (start_line >= 1, end_line >= start_line)
    4. No oversized chunks (> 512 token estimate)
    5. No duplicate chunk IDs
    6. Chunk type distribution (how many function/class/method/window)
    7. Language distribution
    8. File coverage (how many files are represented)

  If all checks pass: "✓ CHUNKS ARE READY FOR WEEK 3 EMBEDDING"
  If issues found:    "✗ CRITICAL ISSUES FOUND" with details on what to fix


  ┌──────────────────────────────────────────────────────────────────────────┐
  │  STEP 4: Run pytest tests                                                │
  └──────────────────────────────────────────────────────────────────────────┘

  pytest tests/ -v

  What it tests:
    tests/test_repo_parser.py   — URL parsing edge cases
    tests/test_chunker.py       — Python AST, JS heuristic, fallback, metadata

  Expected output:
    tests/test_repo_parser.py::TestParseGithubUrl::test_standard_https_url PASSED
    tests/test_chunker.py::TestPythonFunctionExtraction::test_extracts_one_function PASSED
    ...
    35 passed in 0.8s

  If any test fails:
    Read the assertion error carefully.
    It will point to the exact function and expected vs actual value.

WHAT EACH FILE DOES (WEEK 2 ADDITIONS)
────────────────────────────────────────────────────────────────────────────────

utils/file_utils.py
  A utility module for reading and writing chunk data.
  Key functions:
    ensure_dir(path)         — create directory if missing (safe to call twice)
    save_jsonl(chunks, path) — write list of dicts as JSONL (one dict per line)
    load_jsonl(path)         — read JSONL back into list of dicts
    save_json(data, path)    — write dict/list as human-readable JSON
    sanitize_repo_name(o, r) — "my-org", "my-repo" → "my_org__my_repo"
    get_chunk_file_path(o,r) — returns standard JSONL path for a repo

scripts/export_chunks.py
  The main Week 2 command. Full pipeline: fetch → chunk → save JSONL.
  Accepts --max-files flag for large repos.
  Prints full statistics (chunk type breakdown, language breakdown, sizes).

scripts/validate_chunks.py
  Reads a JSONL chunk file and runs 8 quality checks.
  Tells you if chunks are ready for Week 3.
  Returns exit code 1 if critical issues found (useful for CI/CD later).

tests/test_repo_parser.py
  15 pytest test cases for parse_github_url() and make_collection_name().
  Covers normal URLs, edge cases (trailing slash, .git), and invalid inputs.

tests/test_chunker.py
  25 pytest test cases covering:
    - Python function/class/method extraction
    - JavaScript heuristic chunking
    - YAML/text fallback sliding window chunking
    - Required metadata on every chunk type
    - Chunk size limits
    - Edge cases: empty file, syntax error, CRLF line endings

       → embeds them with embedder
       → stores in ChromaDB
       → confirms collection.count() matches chunk count

COMMON ERRORS AND FIXES
────────────────────────────────────────────────────────────────────────────────

ERROR: ModuleNotFoundError: No module named 'pytest'
  FIX: pip install -r requirements.txt  (pytest is now in requirements.txt)

ERROR: FileNotFoundError when running validate_chunks.py
  FIX: Run export_chunks.py first to create the JSONL file.
       python scripts/export_chunks.py https://github.com/psf/requests

ERROR: 0 chunks generated by export_chunks.py
  FIX: All files may have been filtered. Try a repo with Python/JS code.
       Increase --max-files if repo has many files:
       python scripts/export_chunks.py https://github.com/tiangolo/fastapi --max-files 300

ERROR: pytest: command not found
  FIX: Virtual environment is not activated.
       Windows: venv\Scripts\activate
       Mac/Linux: source venv/bin/activate
       Then: pytest tests/ -v

ERROR: RuntimeError / RepoNotFoundError
  FIX: Double-check the repo URL. Is it public? Is the name exact?
       Try: python test_ingester.py https://github.com/psf/requests

ERROR: Validation warns "X chunks oversized"
  FIX: This is a warning, not a critical error. The chunker already applies
       _split_oversized_chunks(), so true oversized chunks should be rare.
       If many are oversized, reduce MAX_CHUNK_CHARS in code_chunker.py.

ERROR: JSONL file created but looks empty
  FIX: Open the file in a text editor. It should have one JSON line per chunk.
       If file size is 0 bytes, the chunker returned 0 chunks.
       Run: python test_ingester.py <repo_url> first to diagnose.

ERROR: Test fails with "AssertionError: repo_name not in chunk"
  FIX: Make sure you are calling chunk_file(..., repo_name="something")
       or chunk_all_files(files, repo_name="owner__repo").
       The test checks that repo_name is absent when NOT provided.

       from sentence_transformers import SentenceTransformer

       import chromadb
       client     = chromadb.PersistentClient(path="./chroma_db")

  3. Run semantic similarity search:
       results = collection.query(query_embeddings=[query_vector], n_results=5)
